# tokenparity v0.2 spec (draft)

Status: **DRAFT** — frozen at v0.1.0a1 release time (2026-05-29). Open for community input before v0.2 implementation begins.

## Scope additions

### Real adapter implementations (v0.1.1 stepping stone)
- `tokenparity.adapters.protein_real.FoldTokenAdapter` — wraps FoldToken (MIT). Loads pretrained weights; runs CPU inference.
- `tokenparity.adapters.video_real.CosmosTokenizerAdapter` — wraps Cosmos-Tokenizer (Apache-2.0). Loads from HF Hub.
- `tokenparity.adapters.robot_real.LeRobotActionAdapter` — wraps LeRobot tokenizer (Apache-2.0).

Each real adapter respects the `KINETOKEN_REAL_DATA=1` + `allow_real=True` two-factor gate. CI keeps real adapters out of the default test set.

### Axis D: Compression (bits-per-dimension)
- `BPD(d, t) = -mean(log p(z|theta)) / dim(z)` measured under the tokenizer's own native decoder, not cross-domain.
- Per-domain only. Cross-domain ranks only.

### Axis E: Equivariance residual
- For tokenizers operating on inputs with a known symmetry group (e.g. SE(3) for protein, SE(2)-time for video, joint reordering for robot):
- `E(d, t) = mean_g ||encode(g · x) − g · encode(x)||`
- Reports as residual scaled by encode(x) norm.

Both new axes are **rank-only** for cross-domain comparison and are gated by sensitivity bands (analogous to IB k∈{3,5,7}).

### Text + audio extras
- `tokenparity.adapters.text.{tiktoken,sentencepiece}_adapter`
- `tokenparity.adapters.audio.{encodec,soundstream}_adapter`
- Both behind `[text-extras]` / `[audio-extras]` install groups.

### dreamcheck (separate repo)
- World-model dual-tokenizer disagreement gate.
- `pip install` consumer of tokenparity; not part of this repo.
- Tracking issue: TBD when repo created at `hinanohart/dreamcheck`.

## Out of scope for v0.2
- GPU acceleration in core path (stays optional behind extras).
- Federated / online evaluation.
- LLM-as-judge style metrics (scalar fusion stance unchanged).
- Cross-modality token translation (out of scope by design; tokenparity *measures*, not *translates*).

## Honest-marketing constraints (carried from v0.1.0a1)
- Still no scalar "best" tokenizer.
- Still typed 5×5 grid (5 axes × 5 modalities when v0.2 adds text+audio).
- Still no live network call in core path.
- Compression and equivariance axes ship with their own dimensionality / dim-sensitivity caveats analogous to IB.

## Implementation order (proposed)
1. v0.1.0a2 hotfix (NITs from S8/S10 audit backlog)
2. v0.1.1 — text + audio adapters + real protein/video/robot adapters (no axis additions)
3. v0.2 — Axis D (BPD) + Axis E (equivariance), with full sensitivity reporting
